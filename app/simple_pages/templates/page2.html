THE DOCKER PLATFORM

Docker is an open source containerization platform that allows developers to package applications into containers. It allows for development, shipping, and running applications separate from a computer's infrastructure. The isolation and security that Docker provides via its Docker Engine allows multiple containers to run at once on a single host.

In this section I will discuss:

  • Containers
  • Images
  • Docker process & terminal commands

CONTAINERS

Containers are standardized units of software that package up code and all their dependencies for the user so that applications can run quickly and smoothly from one environment to another. They are processes on your machine that are isolated from all other processes on the host machine, and can be run on any O.S. They are the live, runnable instances of an image - more on those below.

Although containers are similar to virtual machines (VMs), they do not contain operating system (O.S.) images. VMs work in a hypervisor environment where each VM has its own "guest O.S." running inside it. Containers share one O.S., the one that is running on the machine itself. Because of this difference containers are faster and more lightweight than VMs.

The IBM Cloud Learn Hub goes on to say, "Containers are made possible by process isolation and virtualization capabilities built into the Linux kernel. These capabilities...enable multiple application components to share the resources of a single instance of the host operating system...". Unlike VMs, containers allow for much more effective allocation of computer resources for applications because the resources of the host computer are not locked out. If one container needs to use more RAM, for example, and a separate container isn't using all of its RAM those resources can dynamically be shifted as needed.

IMAGES

A Docker Image is a file used to execute code in a Docker container. The Docker docs website goes on to say, "An image is a read-only template with instructions for creating a Docker container. Often, an image is based on another image, with some additional customization." Essentially working as a set of instructions to build a container, the Image is the "starting point" when using Docker.

The Image contains code, libraries, and other files needed to make the application run inside its container. They consist of multiple layers that stack on top of each other as they are changed by a user. They correlate back to a specific version of the image, acting as a record of a Container at a specific time-point. Docker images are stored on a cloud repository, such as DockerHub.

DOCKER PROCESS & TERMINAL COMMANDS

Docker Process Example

Although Docker is a functional platform that can easily integrate into a text editor such as Pycharm for ease of use, it also has a variety commands that can be utilized directly via a terminal. Below is a step-by-step guide to accessing and running an application with Docker commands, using this web application as an example:

  1. Ensure that Docker has been installed on your local machine so that the commands provided will operate correctly
  2. From the command line of choice navigate to the desired folder location with < cd > commands, and then enter < docker pull austinwilly14/projectsetupflaskbase:master >
  3. After the process has finished, enter < docker images > to see a list of images that are available on your host machine. If you see the new image listed, you know it worked!
  4. After this has been confirmed use the command line to ensure that you are in the correct directory where the image files are stored on your local machine, utilizing < cd> & < pwd > commands
  5. Next, we will compose and run the container for this newly pulled image. Enter < docker-compose up -d > in the appropriate directory with the docker-compose.yml file. "-d" will ensure the processes take place in the background of your terminal
  6. A few moments will pass while the process completes and runs the new container
  7. Enter < docker ps > to see the list of active containers, which should now include the one that was just composed
  8. If all looks as it should, open your web browser and enter into the URL bar: localhost:80
  9. Voila! This web-based application should now be hosted and visible on your local machine in all its glory!

Docker Terminal Commands

Below is a non-comprehensive list of Docker commands that can be used in a terminal such as Command Prompt, Git Bash, Windows Powershell, or the Ubuntu app:

  • docker version = will show the currently installed version of docker that is being utilized by the user
  • docker, or docker help = will list available docker commands for the user to review
  • docker build = builds docker images from a dockerfile in a specific folder location or a directly-specified location, like GitHub repositories or URLs
  • docker run [image name] = will create a new container from an image, specifically named, that has been pulled from DockerHub. Adding a "-dp" to this line adds new arguments to the command. "-d" will run the container you started in detached mode, or in the background so that it doesn't take up space in your terminal screen. "-p" is used to map the ports for the container image, so "-p 80:80" would map port 80 of the host to port 80 in the container
  • docker stop [container id] = will stop the specifically named container from running
  • docker kill [container id] = will stop the specifically named container from running immediately, not allowing for proper shutdown procedures that "stop" would
  • docker start [container id] = will start a container will all previous changes intact
  • docker-compose = will define, run, and share multi-container Docker applications. Once properly prepared, this single command will create and start all the services in your configuration, or tear them all down. This is specified via "docker-compose up" or "docker-compose down" in the command line
  • docker push [online repository information] = will push an image or repository to an online registry, like DockerHub
  • docker pull [image name] = will pull down the specifically named image from the Docker Repository, DockerHub, to the user's machine
  • docker ps = will show a list of all running containers and their metadata information, such as Container ID, Image that is being used, when it was created, how long it has been active, and what ports the container is mapped to
  • docker ps -a = similar but different to the above, this will show a list of all running and exited containers
  • docker images = will show a list of images on the user's docker platform
  • docker image history [image name] = will produce an output of what makes up the specified image. It will display the command that was used to create each layer of the image
  • docker scan [image name] = will utilize the Snyk vulnerability scanning service to scan your specified image for vulnerabilities
  • docker volume create = will create a volume for the container to persist its data through
  • docker network create = will create a network for multiple containers to communicate with one another
  • docker exec [container id] [user command] = will run a command, specified by the user, in a running container
  • docker tag [image id] = will give a new name to the specified image
  • docker rm [container id] = will remove one or more containers
  • docker rmi [image id] = will remove one or more images
