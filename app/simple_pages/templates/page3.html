PYTHON & PYTEST

According to the organization's website: "Python is an interpreted, object-oriented, high-level programming language with dynamic semantics." In simpler terms, it is a programming language used for object-oriented programming of applications.

As written on the pytest organization website: "Pytest is a framework that makes it easy to write small, readable tests that can scale to support complex functional testing for applications and libraries." In this application we utilize pytest functions to ensure content on the interior pages is accurately being displayed. A visual example of this provided to the right.

This test is designed to check against the navigation links and header bar to make sure that the correct pages inside the application are being linked to, and that the right copy is written for each page on the header nav bar. If anything does not match, Pytest will return an error.

An additional pytest example is provided to the right, here. This test is designed to check the content of the About page and ensures that the H1-headline is written and showing appropriately.

Pytest is extremely useful and important. By testing our projects as they are being developed, in a dependable way, we can ensure the work that we are doing is functional and not detrimental to the overall project code.

For this application I utilized a Simple Pages test file to orchestrate the tests for each page and element (more on Simple Page below). The example images from this test file are shown above, on the right-hand side of the page. The functions written in this file scan the pages of the application that are specified and then cross-check the content therein to look for 100% matches. If there is not a 100% match, running a pytest will return errors that then need to be addressed.

FLASK

According to the Flask website: "Flask is a lightweight web application framework designed to make getting started quick and easy." It is a Python module, which means that the entire framework is written in Python code. The repository for this application is hosted on GitHub here. Additionally, here are the Heroku links for the Production and Development Environments for this application.

Flask Blueprints encapsulate functionality for the web app, such as views and templates. It is a very helpful way to organize and work with an application. For this application, I utilize Flask Blueprints for the bundle of controllers that handle routes within the app itself. As you can see in the image to the right, this application uses Flask to import a Blueprint template that is then used to make the bundle of controllers that allow for correct routing between pages.

Simple Pages is also a bundle of controllers made from a Flask Blueprint! With Simple Pages, I have the ability to make template folders that then "speak to one another" efficiently. For this app, we utilize this method to have all the html pages of the site return and route correctly by having a parent "base.html" file living within the root of the app. Any of the internal pages will return correctly in this manner without having to add additional code.

PROJECT FILE BREAKDOWN

  • IS601 Project setup = root of the project; contains all files for the application
    • .github = folder that contains the workflow files for GitHub Actions
      • dev.yml = YAML file that dictates the event(s) and parameters of the Development Workflow in GitHub Actions
      • prod.yml = YAML file that dictates the event(s) and parameters of the Production Workflow in GitHub Actions
    • .pytest_cache = historical records of tests run on this application during pytest
    • app = application folder that houses each of the application .html pages and their content/imagery, as well as appropriate controllers
      • simple_pages = package of templates that are the individual .html pages for this application, and their Flask Blueprint controller bundle
      • static = folder that houses the css and javascript code files for the application, as well as any images that are stored for use on the internal pages
      • templates = houses the base.html file that runs the simple-pages template of header navigation so that it can persist across all pages of the application
      • __init__.py = file that creates and configures an instance of the Flask application
      • run.py = python file that allows Gunicorn to serve the application in production
    • calculator = houses calculator controller file; not in-use for this application at the moment
    • tests = folder that houses all the tests, and their designed parameters for triggering/running, in the application
    • .coveragerc = controller for coverage.py file
    • .dockerignore = file that tells the docker CLI to exclude files and directories that match ".git/", in this application, when building the application image
    • .gitignore = file that tells Git a list of intentionally untracked files to ignore during its processes
    • .pylintrc = file that instructs the pylint check process to ignore certain checks
    • docker-compose.yml = Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
    • Dockerfile = file that is used to create an image to run in a container that runs the program
    • heroku.yml = YAML file that dictates the Docker build for Heroku when hosting the application in Development or Production environments - defines how to build the Docker images on Heroku
    • pytest.ini = configuration file used to hold pytest configuration information - where the test parameters can be pulled from and what parameters to ignore
    • readme.md = Read Me file for the application - text/image/link information that a user can place within the application for context and instructions
    • requirements.txt = Text file that details what python packages are needed to run this application
    • setup.py = Python file that is the build script for "setuptools". Helps to install needed python packages for the application
