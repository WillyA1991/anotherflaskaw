OBJECT ORIENTED PROGRAMMING (OOP) TERMS

  • INSTANTIATION = techopedia defines Instantiation as, " instantiation (the noun) in computer science refers to the creation of an object (or an “instance” of a given class)...an instantiated object is named and created, in memory or on disk."
    • Instantiation is the process of taking a class definition and creating an object that then can be used in a program
    • An Instance is an object that is built from a class and that contains real data
  • CLASS = GeeksforGeeks.com defines a class as a collection of objects - they contain the blueprints from which objects are then created. They are logical entities that can be called upon in the program, and contain methods and attributes
    • A visual example from my code is to the right
  • OBJECT = GeeksforGeeks.com defines object as any single integer, or any single string. Lists, arrays, and dictionaries are all also objects even though they contain other objects
    • they are entities that have state, identity, and behavior associated with them
    • An example is to the right: The test is instantiating the class "Division" from the calculations file. The object "calculation" is then created, from which the test can then run its assertion against
  • NAMESPACE = Namespaces are the files and folders of classes. realpython.com elaborates on this by saying, "A namespace is a collection of currently defined symbolic names along with information about the object that each name references. You can think of a namespace as a dictionary in which the keys are the object names and the values are the objects themselves. Each key-value pair maps a name to its corresponding object."
    • In python there are 4 types of namespaces:
      • Built-In - contains the names of all Python's objects that are available whenever it is running [ex: max(), len(), int(), str()]
      • Global - contains any names defined at the level of the main program. These are created when the program starts and they remain in existence until interpreter is terminated
      • Enclosing - functions can can also define other functions inside themselves. Enclosing namespaces define a function to a local namespace within themselves. When the enclosing namespace is called, the local namespaces will be activated in whatever way they are programmed
      • Local - functions can can also define other functions inside themselves. The namespaces that live inside of an enclosed namespace are called the local namespace. They are called when their enclosing namespace is called
    • As a program runs namespaces are created as necessary and then deleted when no longer needed
    • Different namespaces can have different instances of the same name active, i.e. "def f()" appearing in 2 different namespaces, but this can never be so within a single namespace
    • As long as each instance is in a different namespace, they're all maintained separately and won't interfere
  • CONSTRUCTOR = Constructors are used for instantiating an object. GeeksforGeeks says, "The task of constructors is to initialize(assign values) to the data members of the class when an object of the class is created. In Python the __init__() method is called the constructor and is always called when an object is created."
    • An example is to the right: when the class "Calculation" is instantiated the constructor method shown "(def __init__(self, tuple_list: tuple):" will always get called as the object is being created
  • FIXTURE = Fixtures are functions which will run before each Pytest test to which it is applied. tutorialspoint.com says, "Fixtures are used to feed some data to the tests such as database connections, URLs to test and some sort of input data. Therefore, instead of running the same code for every test, we can attach fixture function to the tests and it will run and return the data to the test before executing each test."
    • A function is marked as a fixture like this: "@pytest.fixture"
    • An example is to the right
  • TYPE HINT = pythontutorial.net states that Type Hinting is a function of the static and dynamic typing that is available in Python. It means that we can hint at the specified input or output of a function
    • The interpreter completely ignores type hints, so if a non-hinted at type is passed through the function will still continue to run
    • An example is to the right: "tuple_list:" is type-hinting that it should be a tuple
  • TYPE CAST = Type Casting is when a variable is directly influenced on what type it will be as it is being run in the program. This can be done with Constructor functions such as:
    • int() - constructs an integer number, given that the input can represent a whole number, str() - constructs a string, and float() - constructs a float number, given that the input can be a float number
    • GeeksforGeeks elaborates on this with "Type Casting is the method to convert the variable data type into a certain data type in order to the operation required to be performed by users."
    • An example here: in the for-loop, each item is being type-cast as "(float(item))"
  • UNIT TEST = Performancelab defines Unit Testing as "testing that involves testing individual components of the software program or application. The main purpose behind this is to check that all the individual parts are working as intended. A unit is known as the smallest possible component of software that can be tested. Generally, it has a few inputs and a single output."
    • This is basically making sure that each of the smallest and smaller bits of the program work, so that we can ensure that the larger program will work, too
    • Having unit tests can also be a great way for someone to easily pick up your project if you leave a team or need additional support, as they will review and test each small component of a program
  • STATIC = Statics are classes that cannot be instantiated, so no objects are going to be created from them. The class members are simply accessed by calling the class name
    • Instantiation is not needed, as the class can simply be called on its own
    • An example is here: this class, "Division", in the operations file
  • STATIC METHOD = are methods that take neither a self or a class (cls) parameter - therefore they are unable to modify an object or class state. They are universally available and do not require instantiation to work
    • According to realpython.com static methods can be called on an object instance but can neither access the object instance state nor the class state. They work like regular functions, but belong to the class' namespace
    • An example is to the right
  • INSTANCE METHOD = these methods have one default parameter that is essential: self. When an object is passed through the Instance Method, it is assigned "Instance Attributes", which GeeksforGeeks says are not shared amongst objects - each object will have its own instance attributes
    • Instance methods enact upon themselves the object parameter that is passed through when the class is called
    • In the example to the right, we see in the "Calculation" class, the instance method "def __init__(self, tuple_list: tuple):"
      • When "Calculation(X)" is called, the self attribute will auto-assign whatever "X" is to that instance attribute
  • INSTANCE PROPERTY = Instance properties are the variables that are created in a class when an Instance Method is called
    • Appearing as variables, they are instead properties because they are created within and a part of a class when it is called
    • In the example to the right, the property is the result and return for "self.values"
  • STATIC PROPERTY = Static properties are the variables that are created in a class when a Static Method is called
    • Appearing as variables, they are instead properties because they are created within and a part of a class when it is called
    • In the example to the right, the property is the result and return for "calculation"
  • ENCAPSULATION = As written on freecodecamp.org "Encapsulation is achieved when each object keeps its state private, inside a class. Other objects don’t have direct access to this state. Instead, they can only call a list of public functions — called methods."
    • This is the idea of wrapping data and the methods that work on that data within one unit
    • This puts restrictions on accessing variables and methods directly, and can prevent the accidental modification of data - to prevent accidental change an object’s variable can only be changed by an object’s method, which is the guiding principle behind "private methods"
    • A class is an example of Encapsulation, as it encapsulates all the data for its member functions, variables, etc
    • In the example to the right Encapsulation is seen with the class "Subtraction" as all the data and methods for that class to be invoked are within the class
  • INHERITANCE = GeeksforGeeks states that Inheritance is "the capability of one class to derive or inherit the properties from another class. The class that derives properties is called the derived class or child class and the class from which the properties are being derived from is called the base class or parent class."
    • Child classes are able to use, or call upon, their parent class methods as though they were their own
    • Child classes can also modify the behavior of the parent class if they are set up to do so
    • In the example to the right, the child class is the "class Division" and it is inheriting from the parent class "(Calculation)"
  • POLYMORPHISM = freecodecamp.org states that, "polymorphism gives a way to use a class exactly like its parent so there’s no confusion with mixing types. But each child class keeps its own methods as they are."
    • This concept makes it possible for the program to use a child class method that is exactly like its parent, but not confuse which is the right one to call for the desired output
    • The child class still has access to the parent method, but changes it as needed for the specific need of the method being called at that point in time
    • In the example to the right, Polymorphism can be seen via the methods "get_result(self)" That method is available for both "Addition" and "Multiplication", but if called the return output will be correct for whatever one was called
      • They are both inheriting from their parent class "(Calculation)"
